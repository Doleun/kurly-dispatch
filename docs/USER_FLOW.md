# USER FLOW — 1차 개발

## 인증 · 권한

```
로그인
  → [통합 관리자] 센터 선택(전체/특정) 후 모든 데이터 접근
  → [센터 담당자] 담당 센터만 자동 필터, 다른 센터 접근 불가
  → 대시보드
```

## 데이터 입력 순서 (권장)

```
0. 센터 설정 (대구/울산) — Phase 1b ✅
1. 관리자 계정 (통합 / 센터 담당자) — Phase 1b ✅
2. 구역 등록 — 센터 → 구역번호 (`20-1` / `20-1가`) — Phase 1b ✅
3. 기사 등록 — 센터, 실명, 컬리 ID, 1차/2차, 고정 구역·랜덤 후보, 기본 ID 매칭 — Phase 1b ✅
4. 휴무 입력 — Phase 1c
5. 배차 — Phase 1d
```

## 기사 등록 (Phase 1b ✅)

```
기사 관리 → [기사 · 구역] 탭
  → 센터 (1개 고정)
  → 계정 유형: ○ 일반  ○ 예비 ID (spare)
  → 실명 (정산용)          예: 홍길동
  → 컬리 아이디 (선택)     예: 300PTH1001
  → 1차/2차 (기사 속성)
  → 고정 구역 (복수) 또는 ☑ 랜덤 후보
  → 최대 처리량
  → [저장]
```

- **예비 ID**: kurly_id 있음, **고정 구역·출근 대상 아님**, 비상 시만 배차표에서 선택
- **2차→1차 지원** 등 당일 변경은 **배차(Phase 1d)** 에서 처리 (기사 관리 아님)
- **목록 클릭 → 수정**, 삭제만 버튼
- **왼쪽 폼 고정 · 오른쪽 목록만 스크롤** (SplitEditorLayout)

## 기본 ID 매칭 (Phase 1b ✅ — 편의용)

```
기사 관리 → [기본 ID 매칭] 탭
  → 실배송자: 홍길동 (kurly_id 없음)
  → 빌릴 ID: 이순신 (300PTH…)
  → 저장 → 배차 초안 **기본값**으로만 반영
```

- **대구·울산 모두** 배차표에서 **당일 ID 변경** 가능
- 50명 중 5명만 다른 ID → 규칙 등록해 두면 편함. 그날 바뀌면 배차에서 수정

## 구역 등록 (Phase 1b ✅)

```
구역 관리
  → 센터 선택
  → 구역번호 한 줄 (20-1 또는 20-1가)
  → 저장
  → 목록 클릭 → 수정
```

- **왼쪽 폼 고정 · 오른쪽 목록만 스크롤**

## 휴무 캘린더 · 주간 휴무표 (Phase 1c — 준비중)

**목표:** 스크린샷과 같은 **주간 휴무 스케줄** (요일별 고정 휴무 + 당월 배치, 1차/2차 분리).

```
휴무표 화면
  → 센터 선택
  → [선택] 그룹별 행: 대구지입 / 대구화성 / 구미지입 / 구미화성 … (센터 설정)
  → 1차 | 2차 탭(또는 상·하단 표)
  → 가로: 요일(월저녁-화새벽 …) · 세로: 기사
  → 셀: 출근/휴무, 구역·ID 참고 (시트와 유사)
  → CAPA·여분 ID·장기휴무 등 사이드 요약 (후순위)
```

### 기사 그룹 (선택 기능)

| 축 | 예 | 비고 |
|----|-----|------|
| **커버 권역** | 대구 / 구미 (대구 센터), 울산 / 부산 (울산 센터) | 센터 1개가 여러 권역 담당 |
| **고용형** | 지입 / 화성(알바) | 휴무표 좌측 라벨 |

- 센터마다 **그룹 사용 ON/OFF** 가능 — OFF면 기사 flat 목록
- Phase 1b `drivers`에 그룹 필드 추가 예정 (1c 착수 시)

## 매일 아침 배차 흐름

```
센터 선택 (담당자는 자동 고정)
  ↓
출근 현황 → 1차/2차 출근자 산출
  ↓
배차 초안 생성 (구역→ID + ID 빌림 규칙 반영)
  ↓
[표 1] 구역→컬리 ID 조정 (매일 변경 가능)
  20-1 → 홍길동 ID,  20-2 → 이순신 ID …
  ↓
[표 2] ID 일일 분할 (정산용 — 구역과 별도)
  홍길동 ID  하루 90건 → A 30 / B 30 / C 30
  이순신 ID  하루 45건 → 홍길동 45 (ID 빌림)
  ↓
※ 컬리 사이트: 표1로 구역별 ID 입력, 표2 total 참고해 ID별 총량 입력 (수동)
  ↓
확정 저장
```

## ID 일일 분할 UX (Phase 1d)

```
[탭: ID 일일 분할]

컬리 ID: [홍길동 ▼]  300PTH1001
하루 전체: [90] 건  ← 컬리에 이 ID로 넣을 총량 참고

실배송자 분할:
  [+ 추가]  A 기사   30건
  [+ 추가]  B 기사   30건
  [+ 추가]  C 기사   30건
  합계: 90건 ✓

[저장]
```

- **구역 배정 화면과 분리** — ID 하루 물량만 여기서 관리
- 구역은 **표 1**에서 매일 따로 지정

## ID 배정 패턴 요약

| 패턴 | 표 1 (구역→ID) | 표 2 (ID 일일 분할) |
|------|---------------|---------------------|
| 본인 ID 1명 | 20-1→홍길동 ID | 홍길동 90 → 홍길동 90 |
| ID 빌림 1:1 | 20-1→이순신 ID | 이순신 pool → 홍길동 90 |
| 1 ID N명 | 여러 구역→홍길동 ID | 홍길동 90 → A30 B30 C30 |
| 예비 ID | 20-1→아무개(spare) | 아무개 pool → 실배송자 |

> **구역은 매일 바뀔 수 있음.** ID 분할은 **ID 하루 total** 기준.

## 정산 (Phase 2 — 흐름만)

```
배차 확정 이력 (actual_driver_id)
  → 월별 정산 집계 (실배송자 기준)
  → 금액·상태 관리
```

Phase 1d까지 **kurly_daily_splits에 실배송자·몫 저장** → Phase 2 정산 연결
